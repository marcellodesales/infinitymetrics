High-Level specs (incomplete and from main documentation)

Users shall have to register and then login in order to use the system Access to the system shall be via Web – Browsers (e.g. Firefox, Netscape or Internet Explorer). System shall have easy to use browser-based UI that allows successful interaction with no prior training for users with low computing skills. On-line help shall be provided (only for one selected function) Internet connection available to customers is expected to be high speed (DSL or cable modem). System shall respond to user query in less than 3 seconds. When entering account information, users shall be connected via secure browser connection High-Level specs – User functions Basic functions available to the user shall be as follows. Those marked * are required.

  • Registration*
  • Login for registered users*
  • Create participation suite*
  • Modify participation suite*
  • Browse for projects*
  • Browse for categories*
  • Search for projects*
  • Search for categories*
  • Select view (tabular or graphical)
  • Output delimited files*

The system has to be implemented in a flexible way. Specifically the collection mechanism should be made modular so that future releases of the java.net infrastructure that provide Web Services APIs can be easily incorporated at a later date.

Integration into real world project The best project will be a candidate for integration into the Education and Research Community at java.net. It will be made available to a growing number of university instructors who will incorporating the teamwork to achieve their pedagogical goals.
