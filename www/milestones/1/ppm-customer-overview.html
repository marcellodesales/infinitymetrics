Basic Overview of Online Project Participation Metrics (PPM) Service :

 

The idea of the project is to develop a web accessible online project participation metrics reporting service for instructors using java.net as a collaborative platform for teaching, training  and managing teamwork in academia or industry. This is very important application for educators and software managers since teamwork during software development is increasingly becoming one of the key skills required for students to be competitive in a global development environment.

 

Within the framework of group collaboration on a common project, various events are considered “participatory events” such as postings to code base and version control systems, to forums etc.  Those events are defined such as to measure teamwork and collaborative activity. The idea is to automate collection of such events by integrating this in the collaborative tools, and to make it easy for instructors and managers to monitor, view and analyse them. This project is specifically targeting java.net as a collaborative platform.    These types of events that require action of a project member at the project site are announced via an RSS feed.  Other participatory events, such as escalated discord within the project team are collected off line by instructors and must be entered into the system manually via easy to use user interface. The service would allow instructors using java.net to set up a project participation metrics reporting suite at the beginning of an academic term.  The PPM service would then periodically monitor participatory events, collect those events and then store them in an online database.  Additionally, instructors should be able to set up additional event categories and then add events manually as they occur.

 

The instructors should be able to initiate a report on the various projects any time after the suite has been created.  Instructors should be able to modify the suite throughout the academic term.  Reports should be provided in two formats, graphical and tabular online and tabular, delimited output files that can be imported into a spreadsheet program.

 

A desktop version of a participation metrics collection system that is currently used by java.net will be provided as a starting point.  That system is written in java and upon launch collects RSS feeds announcing participation events.  Events are then outputted in a delimited file suitable for importing into a spreadsheet.  That system will need to be re-factored into a three-tier Internet application that stores events in an online database.  Guidance from original author will be available.

Users of the system are instructors teaching collaborative project development using java.net as a development platform. Their level of computing knowledge should be considered low but they are familiar with using basic Web and office applications (word processing etc.). They do not necessarily understand internals of Web and database applications. They will not be trained in using this system; hence they expect considerable ease of use and an intuitive interface.

The Online PPM Service should be used in the university or at home, via WWW. The users of the system are assumed to have high speed Internet connection (DSL or cable modem).

 

 

Specifications

 

We will list some high-level preliminary specifications. They are initial specifications, sometimes vague and incomplete, and sometimes quoted as mandatory.  Each team, as part of their Software Engineering training, has to review and revise these requirements if necessary, decide which features are implemented in the first version of their software and which one are options for later versions (in consultation with the instructors and professional observers). The final decisions have to be reviewed by instructors (i.e. management).

High-Level specs (incomplete)

·     Users shall have to register and then login in order to use the system

·     Access to the system shall be via Web – Browsers (e.g. Firefox, Netscape or Internet Explorer).

·     System shall have easy to use browser-based UI that allows successful interaction with no prior training for users with low computing skills.

·     On-line help shall be provided (only for one selected function)

·     Internet connection available to customers is expected to be high speed (DSL or cable modem).

·     System shall respond to user query in less than 3 seconds.

·     When entering account information, users shall be connected via secure browser connection

 

 

High-Level specs – User functions

 

Basic functions available to the user shall be as follows. Those marked * are required.

The system has to be implemented in a flexible way.  Specifically the collection mechanism should be made modular so that future releases of the java.net infrastructure that provide Web Services APIs can be easily incorporated at a later date.

 

 

Integration into real world project

 

The best project will be a candidate for integration into the Education and Research Community at java.net. It will be made available to a growing number of university instructors who will incorporating the teamwork to achieve their pedagogical goals.
